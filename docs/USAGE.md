# debaterAI 사용 가이드

## 1. 설치

```bash
git clone https://github.com/sjk0503/debaterAI.git
cd debaterAI
npm install
npm run dev
```

## 2. 초기 설정

앱 실행 후 우측 상단 **⚙️ Settings** 클릭.

### AI 설정

#### Claude (Anthropic)
- **API Key**: Anthropic Console에서 발급 (`sk-ant-...`)
- **OAuth**: Claude Max 구독이 있으면 OAuth 로그인으로 무료 사용 가능
- **모델 선택**:
  | 모델 | 특징 | 추천 용도 |
  |------|------|-----------|
  | Claude Opus 4 | 최고 성능 | 복잡한 아키텍처 설계, 대규모 리팩토링 |
  | Claude Sonnet 4 | 균형 (기본값) | 일반 코딩, 기능 구현 |
  | Claude Haiku 3.5 | 빠른 응답 | 간단한 수정, 코드 리뷰 |

#### Codex / GPT (OpenAI)
- **API Key**: OpenAI Platform에서 발급 (`sk-proj-...`)
- **OAuth**: ChatGPT Plus/Pro 구독으로 연결 가능
- **모델 선택**:
  | 모델 | 특징 | 추천 용도 |
  |------|------|-----------|
  | GPT-4.1 | 최신 플래그십, 1M 컨텍스트 | 대규모 코드베이스 분석 |
  | GPT-4o | 멀티모달 균형 (기본값) | 일반 코딩, 리뷰 |
  | GPT-4o Mini | 경량 고속 | 간단한 작업 |
  | o3 | 추론 특화 | 복잡한 알고리즘, 로직 설계 |
  | o3-mini | 추론 경량 | 논리적 문제 해결 |
  | o1 | 1세대 추론 | 수학/과학 문제 |

### 고급 설정

- **Temperature**: 0.0~1.0 (낮을수록 일관성, 높을수록 창의성). 코딩은 0.2~0.4 추천.
- **Max Tokens**: 출력 최대 길이. 기본 8192. 큰 파일 생성 시 16384~32000.
- **System Prompt**: AI의 기본 역할 지시. 커스터마이징 가능.

## 3. 토론 모드

### Auto 모드 (기본)
AI끼리 자동으로 토론 → 합의 → 코드 생성.
사용자는 결과만 확인하고 승인/거부.

### Guided 모드
매 라운드마다 사용자가 방향을 결정.
- "Claude 의견 채택"
- "Codex 의견 채택"  
- "토론 계속"
- 직접 의견 입력

### Watch 모드
토론 전체를 관전한 후, 최종 결과에 대해 승인/거부.

### Solo 모드
토론 없이 하나의 AI만 사용.
- **Claude Only**: Claude만으로 코딩
- **Codex Only**: Codex만으로 코딩

## 4. 기본 사용법

### 4.1 프로젝트 열기
1. 하단 📁 입력란에 프로젝트 경로 입력
2. 또는 "폴더 선택" 버튼으로 다이얼로그에서 선택
3. 왼쪽 파일 탐색기에 프로젝트 구조가 표시됨

### 4.2 토론 시작
1. 하단 입력란에 명령 입력: `"JWT 기반 로그인 기능 만들어줘"`
2. "⚔️ 토론 시작" 클릭 (또는 Enter)
3. Claude가 구현 제안 → Codex가 리뷰 → 토론 진행
4. 합의 도달 시 코드 자동 생성

### 4.3 코드 확인 및 적용
1. 토론 패널에서 생성된 코드 확인
2. 오른쪽 코드 뷰에서 파일별 코드 미리보기
3. "적용" 버튼으로 프로젝트에 파일 쓰기
4. Git diff로 변경 사항 확인

### 4.4 명령 예시

```
# 기능 구현
"사용자 인증 기능 만들어줘. JWT + refresh token"
"대시보드 페이지 만들어줘. 차트 3개 포함"
"REST API CRUD 엔드포인트 만들어줘"

# 리팩토링
"이 파일 리팩토링해줘. 중복 코드 제거하고 타입 추가"
"컴포넌트를 hooks로 분리해줘"

# 버그 수정
"로그인이 안 되는 버그 찾아서 고쳐줘"
"TypeScript 에러 전부 수정해줘"

# 코드 리뷰
"이 PR 코드 리뷰해줘"
"보안 취약점 있는지 확인해줘"
```

## 5. Git 연동

### 5.1 워크트리 (Worktree)
각 토론은 격리된 Git 워크트리에서 진행됩니다.

```
프로젝트/
├── .git/
├── src/
└── ...

../.debaterai-worktrees/
├── a1b2c3d4/   ← 토론 #1 (로그인 기능)
├── e5f6g7h8/   ← 토론 #2 (대시보드)
└── ...
```

- 토론끼리 서로 영향 없음
- 완료 시 메인 브랜치에 머지
- 실패 시 워크트리만 삭제 (원본 무영향)

### 5.2 자동 커밋
Settings > Git > Auto Commit 활성화 시:
- 코드 적용할 때 자동으로 `debaterai: [토론 요약]` 커밋
- 커밋 prefix 커스터마이징 가능

### 5.3 브랜치 관리
- 토론마다 `debate/[id]` 브랜치 자동 생성
- 완료 시 현재 브랜치에 `--no-ff` 머지
- 머지 후 debate 브랜치 자동 삭제

## 8. FAQ

### Q: 토론 없이 바로 코딩하고 싶어요
Settings > Debate > Solo Mode에서 "Claude Only" 또는 "Codex Only" 선택.

### Q: 토론이 너무 오래 걸려요
Max Rounds를 2로 낮추거나, 빠른 모델(Haiku, GPT-4o Mini) 사용.

### Q: API 비용이 걱정돼요
- Claude: Max 구독 ($100/월)이면 OAuth로 무료
- OpenAI: 4o-mini 사용 시 매우 저렴
- Temperature 낮추면 재시도 줄어서 비용 절약

### Q: 프로젝트가 너무 커서 컨텍스트가 부족해요
- Claude Opus 4 (200K) 또는 GPT-4.1 (1M) 사용
- Max Files 설정으로 컨텍스트에 포함할 파일 수 조절

### Q: Git 워크트리가 뭐예요?
같은 저장소의 다른 브랜치를 별도 폴더에 체크아웃하는 기능.
토론마다 독립 환경을 제공해서 충돌 없이 병렬 작업 가능.
